OCR Classifier - Эндпоинт /ocr-classifier/api/v1/classify

Описание

Эндпоинт /ocr-classifier/api/v1/classify предназначен для классификации изображений на наличие текста с использованием Tesseract OCR. Сервис анализирует изображение и возвращает детальную информацию о найденном тексте, включая показатели уверенности (confidence), координаты текстовых фрагментов и количество найденных токенов (текстовых символов, несущих смысловую нагрузку). Фрагменты текста с уверенностью распознавания ниже 0.25 отфильтровываются и не учитываются.

HTTP-запрос

POST /ocr-classifier/api/v1/classify[?confidence_threshold=0.66&min_token_count=20&lang=eng+rus&level=RIL_WORD]
Content-Type: image/jpeg | image/png
Body: <бинарные данные изображения>

URL-параметры

Параметр Тип По умолчанию Описание
lang string eng+rus Языки для Tesseract OCR. Поддерживаются любые языки, установленные в системе. Несколько языков разделяются плюсом (например, eng, rus, eng+rus, deu+fra). По умолчанию используется комбинация eng+rus.
level string RIL_WORD Уровень детализации структуры текста (PageIteratorLevel). Допустимые значения: RIL_BLOCK (блок), RIL_PARA (абзац), RIL_TEXTLINE (строка), RIL_WORD (слово, по умолчанию), RIL_SYMBOL (символ). Более детальные уровни (например, RIL_SYMBOL) могут значительно увеличить время обработки.
confidence_threshold float64 0.66 Минимальная взвешенная уверенность (0.0 - 1.0) для вердикта "текстовый документ". При достижении порога вместе с min_token_count дальнейшие попытки OCR прекращаются.
min_token_count int 20 Минимальное количество токенов для вердикта "текстовый документ". При достижении порога вместе с confidence_threshold дальнейшие попытки OCR прекращаются.

Поддерживаемые форматы

  • image/jpeg - изображения в формате JPEG
  • image/png - изображения в формате PNG

Параметры ответа

Параметр Тип Описание
mean_confidence float64 Средняя уверенность распознавания по всем текстовым блокам (0.0 - 1.0)
weighted_confidence float64 Взвешенная уверенность с учетом количества токенов в каждом блоке (0.0 - 1.0). Более точный критерий для определения "текстового документа", чем mean_confidence.
token_count int Общее количество распознанных токенов (буквы, цифры и спецсимволы, несущие смысловую нагрузку). Токены во фрагментах с confidence ниже 0.25 не учитываются.
boxes array Массив текстовых блоков с координатами и содержимым
angle int Угол поворота изображения, определенный алгоритмом deskewing (0-359 градусов)
scale_factor float64 Коэффициент масштабирования, примененный к изображению
is_text_document boolean Вердикт: является ли документ текстовым (true если weighted_confidence >= confidence_threshold И token_count >= min_token_count)
bounding_box_width int Ширина изображения в пикселях после применения предобработки (масштабирование и поворот)
bounding_box_height int Высота изображения в пикселях после применения предобработки (масштабирование и поворот)

Структура BoundingBox

Параметр Тип Описание
x int X-координата левого верхнего угла блока
y int Y-координата левого верхнего угла блока
width int Ширина текстового блока в пикселях
height int Высота текстового блока в пикселях
word string Распознанный текст в блоке
confidence float64 Уверенность распознавания данного блока (0.0 - 1.0). Блоки с confidence ниже 0.25 отфильтровываются и не возвращаются.

Коды ошибок

Код Описание
405 Неверный HTTP-метод (только POST)
400 Неверный Content-Type, пустое изображение или ошибка чтения данных
500 Ошибка обработки изображения или Tesseract OCR

Алгоритм обработки

Предобработка изображения

Перед распознаванием изображение проходит несколько этапов предобработки:

  1. Динамическое масштабирование - размер изображения корректируется в зависимости от количества пикселей:
    • < 0.5 MP: увеличение в 4 раза
    • 0.5-1 MP: увеличение в 3 раза
    • 1-2 MP: увеличение в 1.5 раза
    • 2-3 MP: без изменений
    • > 3 MP: уменьшение до 3 MP
  2. Конвертация в градации серого - упрощает анализ и улучшает распознавание
  3. Медианный фильтр - уменьшает шум на изображении

Двухфазное распознавание

Для улучшения качества распознавания используется двухфазный подход:

  1. Фаза 1: OCR без поворота. Если оба критерия вердикта выполнены (взвешенная уверенность ≥ confidence_threshold И количество токенов ≥ min_token_count), результат возвращается сразу.
  2. Фаза 2: Автоматическое определение угла наклона документа (deskewing) с помощью алгоритмов компьютерного зрения:
    1. Canny edge detection — выделение контуров на изображении (Sobel-градиенты, подавление немаксимумов, двойная пороговая фильтрация с гистерезисом).
    2. Hough Line Transform — обнаружение прямых линий в карте контуров. Порог голосования адаптивный, пропорциональный размеру изображения.
    3. Определение доминирующего угла наклона — взвешенная медиана углов 20 наиболее выраженных линий.
    4. Генерация кандидатов — на основе обнаруженного наклона формируется до 4 углов поворота (наклон, наклон+90°, наклон+180°, наклон+270°), исключая 0° (уже проверен в фазе 1). Если линии не обнаружены, используются резервные углы: 90°, 180°, 270°.
    5. Параллельное OCR — пул воркеров (до 4) выполняет распознавание для каждого кандидата с ранним выходом при достижении критериев вердикта.

Вердикт "Текстовый документ"

Сервис определяет, является ли изображение текстовым документом, на основе двух критериев:

  • Взвешенная уверенность ≥ confidence_threshold (по умолчанию 0.66)
  • Количество токенов ≥ min_token_count (по умолчанию 20)

Оба условия должны быть выполнены одновременно. Результат возвращается в поле is_text_document.

Примечание: Поддерживаются языки: английский (eng) и русский (rus).

Подсчет токенов

Токены учитываются по следующим правилам:

  • Латинские и кириллические буквы
  • Цифры
  • Точка, запятая, двоеточие или косая черта между цифрами (например, "3.14", "1,000", "1:00", "1/2")
  • Плюс, минус или Unicode-минус перед цифрой (например, "+5", "-10", "−10")
  • Символы единиц измерения после цифры (например, "45°", "30′", "15″", "5‰", "25℃", "77℉", "1Å", "5µm")
  • Валютные символы перед или после цифры (например, "$100", "50€")
  • Кавычки, соседствующие с буквами или цифрами (например, "abc", '123', «text»)

Примеры использования

Базовый запрос

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  http://localhost:8080/ocr-classifier/api/v1/classify

С пользовательским языком и уровнем детализации

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  "http://localhost:8080/ocr-classifier/api/v1/classify?lang=rus&level=RIL_WORD"

С пользовательским порогом уверенности

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  "http://localhost:8080/ocr-classifier/api/v1/classify?confidence_threshold=0.8"

С пользовательскими критериями вердикта

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  "http://localhost:8080/ocr-classifier/api/v1/classify?confidence_threshold=0.7&min_token_count=15"

Полный пример с всеми параметрами

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  "http://localhost:8080/ocr-classifier/api/v1/classify?lang=eng+rus&level=RIL_TEXTLINE&confidence_threshold=0.75&min_token_count=10"
