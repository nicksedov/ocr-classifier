OCR Classifier - Эндпоинт /classify

Описание

Эндпоинт /classify предназначен для классификации изображений на наличие текста с использованием Tesseract OCR. Сервис анализирует изображение и возвращает детальную информацию о найденном тексте, включая показатели уверенности (confidence), координаты текстовых блоков и количество найденных токенов.

HTTP-запрос

POST /classify[?confidence_threshold=0.66]
Content-Type: image/jpeg | image/png
Body: <бинарные данные изображения>

URL-параметры

Параметр Тип По умолчанию Описание
confidence_threshold float64 0.66 Пороговое значение уверенности (0.0 - 1.0). При достижении этого порога дальнейшие попытки OCR с поворотом изображения прекращаются.

Поддерживаемые форматы

  • image/jpeg - изображения в формате JPEG
  • image/png - изображения в формате PNG

Ответ

200 OK - Успешный ответ

{
  "mean_confidence": 0.85,
  "weighted_confidence": 0.88,
  "token_count": 12,
  "boxes": [
    {
      "x": 10,
      "y": 20,
      "width": 100,
      "height": 50,
      "word": "Example",
      "confidence": 0.95
    }
  ],
  "angle": 0,
  "scale_factor": 1.0
}

Параметры ответа

Параметр Тип Описание
mean_confidence float64 Средняя уверенность распознавания по всем текстовым блокам (0.0 - 1.0)
weighted_confidence float64 Взвешенная уверенность с учетом количества токенов в каждом блоке (0.0 - 1.0)
token_count int Общее количество распознанных токенов (буквы, цифры, спецсимволы)
boxes array Массив текстовых блоков с координатами и содержимым
angle int Угол поворота изображения для лучшего распознавания (0-359 градусов)
scale_factor float64 Коэффициент масштабирования, примененный к изображению

Структура BoundingBox

Параметр Тип Описание
x int X-координата левого верхнего угла блока
y int Y-координата левого верхнего угла блока
width int Ширина текстового блока в пикселях
height int Высота текстового блока в пикселях
word string Распознанный текст в блоке
confidence float64 Уверенность распознавания данного блока (0.0 - 1.0)

Коды ошибок

Код Описание
405 Неверный HTTP-метод (только POST)
400 Неверный Content-Type, пустое изображение или ошибка чтения данных
500 Ошибка обработки изображения или Tesseract OCR

Алгоритм обработки

Предобработка изображения

Перед распознаванием изображение проходит несколько этапов предобработки:

  1. Динамическое масштабирование - размер изображения корректируется в зависимости от количества пикселей:
    • < 0.5 MP: увеличение в 4 раза
    • 0.5-1 MP: увеличение в 3 раза
    • 1-2 MP: увеличение в 1.5 раза
    • 2-3 MP: без изменений
    • > 3 MP: уменьшение до 3 MP
  2. Конвертация в градации серого - упрощает анализ и улучшает распознавание
  3. Медианный фильтр - уменьшает шум на изображении

Двухфазное распознавание

Для улучшения качества распознавания используется двухфазный подход:

  1. Фаза 1: OCR без поворота. Если взвешенная уверенность ≥ confidence_threshold (по умолчанию 0.66), результат возвращается сразу.
  2. Фаза 2: Параллельное тестирование 19 углов поворота (90°, 180°, 270° и отклонения ±5°, ±10° от основных ориентаций). Используется пул из 4 воркеров с ранним выходом при достижении порога уверенности.
Примечание: Поддерживаются языки: английский (eng) и русский (rus).

Подсчет токенов

Токены учитываются по следующим правилам:

  • Латинские и кириллические буквы
  • Цифры
  • Точка/запятая между цифрами (например, "3.14", "1,000")
  • Плюс/минус перед числом (например, "+5", "-10")
  • Символы процента, промилле, градуса, минут и секунд после числа (например, "45°", "30′", "15″", "5‰")
  • Символы валют рядом с числом (например, "$100", "50€")

Примеры использования

Базовый запрос

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  http://localhost:8080/classify

С пользовательским порогом уверенности

curl -X POST \
  -H "Content-Type: image/jpeg" \
  --data-binary @image.jpg \
  "http://localhost:8080/classify?confidence_threshold=0.8"
