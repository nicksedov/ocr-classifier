## Сборка и запуск сервиса *ocr-classifier*

Для сборки и запуска сервиса *ocr-classifier* из определённой ветки кода выполните шаги ниже.

### 1. Подготовка
- Выполнить вход на сервер *tvlew-sb0000001.esrt.sber.ru* под пользователем *podmanuser* (с помощью ssh или putty):  
```bash
ssh podmanuser@tvlew-sb0000001.esrt.sber.ru
```
Все дальнейшие шаги выполняются в открытой ssh-сессии

### 2. Переход в рабочую директорию
- После успешного входа, перейти в директорию:
```bash
cd /opt/gigaoffice/podman/tesseract/fc42
```

### 3. Сборка образа из нужной ветки
Требуется запустить скрипт автоматизации сборки *build.sh*. Скрипт поддерживает передачу имени ветки параметром. Если параметр не указывать, используется ветка *master*.

Примеры:
- Для ветки *feature/abc*:
  ```bash
  ./build.sh feature/abc
  ```
- Для ветки по умолчанию (*master*):
  ```bash
  ./build.sh
  ```

При запуске скрипта выполняются следующие последовательные шаги:

#### Проверка прав
Скрипт проверяет, запущен ли он от имени учетной запси podmanuser. Если нет — выводится сообщение об ошибке и выполнение прерывается.
#### Определение рабочей директории
Вычисляется абсолютный путь к папке, где расположен скрипт, чтобы все дальнейшие операции (в том числе размещение временных файлов) выполнялись относительно этого каталога.
#### Подготовка сертификатов
Создаётся структура каталогов для доверенных сертификатов Сбера (etc/pki/ca-trust/source/), после чего туда копируются необходимые корневые сертификаты из системы.
#### Копирование SSH-ключей
Директория .ssh пользователя (/home/podmanuser/.ssh) копируется в рабочий каталог скрипта. Это обеспечивает доступ к приватным репозиториям при выполнении git clone внутри контейнера.
#### Очистка старых образов и контейнеров
Останавливаются и удаляются все контейнеры, созданные на основе целевого образа.
Принудительно удаляется сам образ. Выполняется очистка системы Podman (удаление неиспользуемых образов, сетей и кэшированных данных).
#### Сборка Docker-образа
Запускается команда сборки образа. На этом шаге наименование ветки передаётся в Dockerfile, где она подставляется в команду git clone.
#### Создание и запуск контейнера
Создаётся новый контейнер с именем ocr-classifier-fc42 и пробрасывается порт 10080 во внутренний 8080.
Контейнер запускается автоматически сразу после создания.
#### Очистка временных файлов
После успешной сборки и запуска удаляются скопированные ранее директории с сертификатами и SSH-ключами, чтобы не оставлять временные данные в рабочем каталоге.
Итого, весь процесс полностью автоматизирован: от подготовки окружения и очистки старых артефактов до сборки, запуска и финальной уборки. Дополнительных ручных действий не требуется.

### 4. Проверка запуска контейнера
Проверить успешный запуск можно командой:
```bash
podman ps
```

### 5. Проверка работоспособности
Сервис доступен на порту 10080. Проверьте статус через endpoint */health*:
Находясь в сеансе ssh:
```bash
curl http://localhost:10080/health
```
Для проверки доступа с удаленного хоста:
```bash
curl http://tvlew-sb0000001.esrt.sber.ru:10080/health
```
Ожидаемый успешный ответ:
```json
{"status":"ok"}
```

### 6. Примеры запросов к API
Файл с примерами *curl*-команд находится по пути:
```
/opt/gigaoffice/podman/tesseract/examples/curl.txt
```
Просмотреть примеры:
```bash
cd /opt/gigaoffice/podman/tesseract/examples && cat curl.txt
```

### 7. Остановка и очистка
При необходимости остановить и удалить контейнер:
```bash
podman stop ocr-classifier-fc42
podman rm ocr-classifier-fc42
```
Перед пересборкой с использованием скрипта build.sh удалять контейнер вручную не требуется.
