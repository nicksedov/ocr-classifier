Умный поиск в облачном хранилище "Диск"

Обзор решения

Система умного поиска позволяет пользователям облачного хранилища "Диск" осуществлять семантический поиск по содержимому загруженных документов. Решение интегрирует несколько компонентов для обработки, классификации и индексации документов различных форматов.

Поддерживаемые форматы документов

Office (DOC, DOCX, XLS, XLSX, PPT, PPTX) PDF Изображения (JPG, PNG)

Ключевые возможности

  • Автоматическая классификация изображений на наличие текстового контента
  • Извлечение текста из документов всех поддерживаемых форматов
  • Семантическая индексация с использованием векторных эмбеддингов
  • Поиск по смыслу запроса, а не только по ключевым словам

Архитектура системы

Система построена на базе платформы GigaSearch IDP и состоит из трех основных модулей:

Data Enrichment

  • OCR распознавание
  • Парсинг файлов
  • Text Splitting (чанкинг)
  • Генерация эмбеддингов
  • Генерация FAQ

Universal Indexer

  • Создание поисковых индексов
  • Обновление индексов
  • Хранение чанков в S3
  • Управление DataVersion

Universal Search

  • Семантический поиск
  • Reranking результатов
  • Генерация ответа через LLM
  • Работа с источниками

Технологический стек

S3 (чанки) Qdrant (векторы) OCR Classifier GigaSearch IDP API GigaChat LLM

Жизненный цикл загрузки данных

1
Пользователь загружает на Диск офисный документ, PDF или изображение (JPG/PNG)
2
Документ сохраняется во внутреннем хранилище Диска
3
Для изображений (JPG/PNG) выполняется бинарная классификация через ocr-classifier /classify — определяется, содержит ли изображение текстовый документ
4
Текстовые документы (Office, PDF, изображения с текстом) отправляются в GigaSearch IDP для распознавания и индексации
5
Сформированные чанки сохраняются в S3, векторные эмбеддинги — в Qdrant

Роль OCR Classifier в процессе загрузки

Микросервис ocr-classifier решает задачу бинарной классификации для изображений: определяет, является ли загруженное изображение фотографией текстового документа (скан, фото документа) или обычной фотографией/картинкой без значимого текстового содержимого.

Это позволяет избежать ненужной индексации изображений без текста и оптимизировать использование ресурсов платформы GigaSearch IDP.

Жизненный цикл умного поиска

1
Пользователь вводит поисковый запрос в свободной форме через UI Диска
2
Система выполняет семантический поиск релевантных документов через GigaSearch IDP API
3
Результаты ранжируются (Reranker) и возвращаются пользователю

Интеграционные точки

OCR Classifier API

Эндпоинт Метод Назначение
/classify POST Бинарная классификация изображения на наличие текстового документа
/health GET Проверка работоспособности сервиса

GigaSearch IDP API

Платформа предоставляет API для:

  • Загрузки данных из различных источников (файлы, FAQ, текстовые массивы)
  • Управления индексами и DataVersion
  • Выполнения семантического поиска
  • Генерации ответов через LLM GigaChat
  • Реализации чат-скиллов поверх индекса
